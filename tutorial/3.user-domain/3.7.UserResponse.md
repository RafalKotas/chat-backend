# 3.7 UserResponse - API Output DTO

`UserResponse` is a **read-only DTO** used to send user information back to the client.

It defines **what data is safe to expose externally**,

---

This DTO is responsible for:

- returning sanitized user data
- hiding sensitive internal fields
- providing a stable API response format
- preventing accidental data leaks


---

## ðŸ“Œ Field Descriptions
### id
- Public unique identifier
- Used for referencing the user in API calls
- UUID ensures global uniqueness

---

### email
- Returned for confirmation purposes
- Useful for frontend account displays
- Never editable directly via API

---

### displayName
- Computed field
- Derived from:
  - username (if present)
  - first + last name
  - fallback identifier
- Used for chat UI and contact lists

---

### createdAt
- Account creation timestamp
- Useful for auditing and UI display
- Read-only value

--- 

## ðŸš« Fields Intentionally Omitted

The following fields are **never exposed:**
- password
- role
- enabled flag
- internal timestamps
- security-related attributes
