# üß± 3.5 User Exceptions ‚Äî Domain-Level Error Handling
This file documents the custom exceptions used in the **User domain**.

These exceptions ensure that the domain logic communicates errors clearly and predicatably, rather than relying on 
generic Java exceptions.

---

# ‚ùó Why Custom Exceptions?
Custom exceptions allow us to:
- express domain rule violations explicitly
- avoid ambiguous errors like `IllegalStateException`
- map failures cleanly to HTTP responses (e.g., **409 Conflict**)
- improve readability in logs
- simplify unit testing
- keep controller/service layers clean and intention-revealing

---

## üìå `EmailAlreadyUsedException`

### When it is thrown

When attempting to register a new user with an email that already exists in the system.

### Why

Email must be unique for:
- login
- account recovery
- identity verification

### Constructor example
```java
public class EmailAlreadyUsedException extends RuntimeException {
    public EmailAlreadyUsedException(String email) {
        super("Email already in use: " + email);
    }
}
```

---

## üìå `UsernameAlreadyUsedException`

### When it is thrown

When a user attempts to register with a username that already exists.

### Why

Although username is optional, if provided it must be unique for:
- profile identification
- chat labeling
- future features (mentions, search)

### Constructor example
```java
public class UsernameAlreadyUsedException extends RuntimeException {
    public UsernameAlreadyUsedException(String username) {
        super("Username already in use: " + username);
    }
}
```
